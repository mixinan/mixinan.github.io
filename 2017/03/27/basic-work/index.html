项目经理的基本工作要求

Mar 27, 2017 | 阅读

一些常识

一次违规处罚

周日下午上班期间，项目经理 WP 睡觉，并且有学员上课玩游戏未被及时发现和制止，影响非常不好。鉴于王攀认错态度很好，本次仅罚款 100 元。

作为教学主管，我没有专门给团队强调过这方面的纪律和要求。因为觉得上班期间不做与工作无关的事情，尤其是玩游戏、闲逛网页、购物、睡觉等，应该是最基本的职业常识。WP 犯错，我也有责任。所以，我为他承担一半，分别从我们本月工资中扣除 50 元。

公司不允许出现的行为，就不要让它发生。处罚不是目的，它只是提醒我们不要犯更大的错误、付出更大代价。

项目经理与串讲

前几天有学员问我，你们如果招聘项目经理，有什么要求？我说：

  1. 技术好：起码要能给学生解决问题、提供帮助；对技术有热情，渴望新知识，并愿意付诸行动。
  2. 负责任：对学生和公司负责，积极主动解决学员的问题；营造学习气氛，有办法引导班级向上。
  3. 口才佳：能扯淡、能讲课、能推销、能维稳，甚至能提高满意度分数。刻意练习提高讲话技能。
  4. 情商够：乐观，耐心，懂得与人沟通，部门合作积极。有想法，愿意寻找解决问题的最佳途径。
  5. 作榜样：我们什么样，达内在学生眼里就什么样；我们什么样，带出来的学生就什么样；学生什么样，达内的品牌形象就什么样。时刻要做学生榜样，配得上“在进班第一天时，他们对我们的期望”。

我不清楚他问这个目的是什么，可能是想当项目经理，也可能是要看看“我们对自己的要求”和“我们的实际情况”是否一致？我不知道。

我是按照“我对自己的期望”来说的，希望自己是这样的人，当然也希望能与更多这样的人共事，那样彼此都会很省心。

从我到南京，一直在新街口中心工作。一年以来，听过几次关于其它中心项目经理的说法，有的是咨询同事告诉我的（学员给她反馈），有的是从别的中心转班来的学员告诉我的，大致都是“项目经理很少讲话”“晚上从来不串讲”“项目经理估计新来的”“项目经理跟不存在一样”等。据此我猜测，我们新街口的同事，和其他中心不一样。

直到去年底，因为开新产品线的需要，我急招了一个已经从外中心毕业两个多月的学员作同事。他的所作所为，让我非常无语——技术能力、自觉性、责任感都很差。简直可以说是“烂泥扶不上墙”。他说自己的项目经理就是这样带班的。一个星期后，我辞退了他。

我感到非常疲惫。其它中心项目经理的工作，真的如我听到的那样吗？

工作不是那样做的。我相信，在任何行业和岗位，收获和付出都是成正比的。所有的努力，总会得到回馈，有的是以金钱，有的是以机会，或者其它能改善现状的方式。爱岗敬业，并非说说而已。工作不是机械式地劳作，作为投入了时间、精力和身体的参与者，我们要去发现其中的价值和意义，从中获得成就感和满足感，这样可能才是幸福的。否则一切都是为了什么？

去年 7 月，南京教学部组织了一次讲课比赛。我们中心 ZZZ 向我吹牛说，以前这种比赛，新街口肯定包揽前三名。我问为啥？他说实力。真是太自大了，我说：“现在就我们几个，还行吗？” 他说“反正我不参加，你们新人上”。最后我和另一个同事参加了，分别拿了第 2、3 名。

第一名是一个带班经验丰富的 Java 女项目经理。还有几个老项目经理，排名在我们之后。当然，这并不表示我们实力就强，或者人家就不行，但至少说明了，我们平时经常主动做的串讲，对自身素质是有提高的。不然我一个从小内向寡言、连新街口都没有出去过的新人，凭什么得到其他中心陌生同事的掌声和分数？

串讲的主要目的，是要解决学生当下遇到的问题，或难以理解的知识点，帮助他们梳理知识。可能有同事以为，把学过的知识点串一下没什么意义。我得认真、明确地说，这种想法大错特错！

串讲做与不做，给学员的感觉是完全不同的。我也是学员出身，当时助教有几次串讲，虽然只是简单过一下当天笔记，但给了我特别大的好感。新知识一旦重复，并且通过我们自己的语言再做简单讲解，会对学生产生非常大的正面作用。因为尝过别人串讲带来的甜头，所以我每天多多少少都会结合自己的理解，简单给他们过一下知识点，希望每个人也都能尝到一样的甜头。当然，也可以讲一些课程里未涉及的其它知识，但要少量。解决眼前的问题，才是重点。

月底可能会有一个新同事加入，对于新团队，我的期望就是上面 5 点，在带好班的基础上，我们一起刻意练习，使每个人每个阶段都能在某一方面有所进步。相信我，一切都不会白费。愿我们所有人严于律己，别再犯错！
